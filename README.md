# b2bflow-python-supabase-zapi

Projeto Python que le contatos cadastrados no Supabase e envia mensagens personalizadas via Z-API (WhatsApp).

Desafio tecnico para o processo seletivo de Estagio em Desenvolvimento Python na b2bflow.

---

## Objetivo

Ler contatos ativos de uma tabela no Supabase e enviar, via Z-API, a mensagem:

> "Ola, <nome_contato> tudo bem com voce?"

---

## 1. Setup da Tabela no Supabase

1. Acesse [supabase.com](https://supabase.com) e crie um projeto gratuito.
2. Va em **SQL Editor** e execute o conteudo do arquivo `sql/setup.sql`:

```sql
CREATE TABLE IF NOT EXISTS contatos (
    id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nome_contato TEXT NOT NULL,
    telefone     TEXT NOT NULL,
    ativo        BOOLEAN DEFAULT TRUE,
    created_at   TIMESTAMPTZ DEFAULT NOW()
);

INSERT INTO contatos (nome_contato, telefone, ativo)
VALUES
    ('Contato Teste 1', '5511999999999', true),
    ('Contato Teste 2', '5511988888888', true),
    ('Contato Teste 3', '5511977777777', true);
```

3. Copie a **URL** e a **Anon Key** do seu projeto em **Project Settings > API**.

---

## 2. Variaveis de Ambiente (.env)

Copie o arquivo de exemplo e preencha com suas credenciais:

```bash
cp .env.example .env
```

Edite o arquivo `.env`:

```env
# Supabase
SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
SUPABASE_KEY=sua_anon_key_aqui
SUPABASE_TABLE=contatos

# Z-API (WhatsApp)
ZAPI_INSTANCE_ID=seu_instance_id
ZAPI_INSTANCE_TOKEN=seu_instance_token
ZAPI_CLIENT_TOKEN=seu_client_token

# Configuracoes
DRY_RUN=true
MAX_CONTACTS=3
```

> O arquivo `.env` esta no `.gitignore` e nunca sera enviado ao GitHub.

## 5. Modos de Execucao

### Modo Teste (DRY_RUN=true)

Simula o envio sem fazer chamadas reais para a Z-API. Ideal para testar a conexao com o Supabase.

```bash
# No .env: DRY_RUN=true
python main.py
```

Saida esperada:
```
2024-01-01 12:00:00 [INFO] Modo: DRY_RUN (simulacao)
2024-01-01 12:00:00 [INFO] Processando contato: Contato Teste 1 (5511999999999)
2024-01-01 12:00:00 [INFO] [DRY_RUN] Simulando envio para 5511999999999: 'Ola, Contato Teste 1 tudo bem com voce?'
```

### Modo Real (DRY_RUN=false)

Envia as mensagens de verdade via Z-API.

```bash
# No .env: DRY_RUN=false
python main.py
```

> Certifique-se de que o WhatsApp esta conectado na sua instancia Z-API antes de executar em modo real.
